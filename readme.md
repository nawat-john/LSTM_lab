# 9arm Text Generator (LSTM Model)

โปรเจกต์นี้เป็นการสร้างโมเดล Machine Learning แบบ Deep Learning ด้วยสถาปัตยกรรม **LSTM (Long Short-Term Memory)** เพื่อทำหน้าที่สร้างข้อความหรือประโยคเลียนแบบสไตล์การพูดของช่องยูทูป "9arm" โดยใช้ข้อมูลจากซับไตเติล (Transcript) ของวิดีโอบน YouTube

## หลักการทำงาน (คร่าวๆ)

การทำงานของไฟล์ `lstm.ipynb` สามารถแบ่งออกได้เป็นขั้นตอนหลักๆ ดังนี้:

### 1. การดึงข้อมูลและการตัดคำ (Data Collection & Tokenization)
- ใช้ไลบรารี `youtube_transcript_api` เพื่อดาวน์โหลดคำบรรยาย (Transcript) จากวิดีโอ YouTube ของช่อง 9arm ตาม `VIDEO_IDS` ที่กำหนดไว้
- หลังจากได้ข้อความทั้งหมดมารวมกันแล้ว ข้อมูลจะถูกนำไปผ่านกระบวนการทำความสะอาดและตัดคำภาษาไทย (Word Segmentation) ด้วยไลบรารี **PyThaiNLP** ในสคริปต์ `process_transcript.py` แล้วบันทึกผลลัพธ์เป็นชุดข้อมูลใน `transcript.csv` เพื่อนำมาใช้ฝึกสอนโมเดลต่อไป

### 2. การเตรียมข้อมูล (Data Preprocessing)
- อ่านข้อมูลประโยคจากไฟล์ `transcript.csv`
- **Tokenization**: แปลงคำต่างๆ ในข้อความให้เป็นตัวเลข (Token) โดยใช้ `Tokenizer` ของ Keras ซึ่งจะสร้างพจนานุกรมคำศัพท์ (Vocabulary) ทั้งหมดขึ้นมา
- **N-gram Sequences**: นำลำดับตัวเลขที่ได้มาสร้างเป็นลำดับคำแบบ N-gram เช่น ใช้คำที่ 1 ทำนายคำที่ 2, ใช้คำที่ 1-2 ทำนายคำที่ 3 ต่อเนื่องไปเรื่อยๆ เพื่อสร้างเป็นรูปแบบการเรียนรู้
- **Padding**: ปรับความยาวของทุก sequences ให้เท่ากัน (Padding) เพื่อให้อยู่ในรูปแบบเมทริกซ์ที่สามารถป้อนเข้าสู่โมเดลได้
- แยกข้อมูลออกเป็นชุดข้อมูลนำเข้า (`X` - กลุ่มคำเริ่มต้น) และชุดข้อมูลผลลัพธ์ (`y` - คำถัดไปที่ต้องการให้โมเดลทำนาย)

### 3. การสร้างและฝึกสอนโมเดล (Model Architecture & Training)
สถาปัตยกรรมโมเดล Keras (Sequential) ประกอบด้วย:
- **Embedding Layer**: แปลงคำ (ที่เป็นตัวเลขดัชนี) ให้กลายเป็นเวกเตอร์ (Vector) ที่สามารถบอกถึงความสัมพันธ์ของคำแต่ละคำได้
- **Bidirectional LSTM Layer**: เลเยอร์หลักแบบ Recurrent Neural Network (RNN) ชนิด LSTM ที่รับข้อมูลทั้งจากทิศทางไปข้างหน้าและย้อนหลัง ช่วยให้จับบริบทความหมายของประโยคได้ดีเยี่ยม และแก้ปัญหาการลืมข้อมูลในอดีต (Vanishing Gradient)
- **Dropout Layer**: ช่วยป้องกันไม่ให้โมเดลจดจำข้อมูลมากเกินไป (Overfitting) โดยการสุ่มปิดการทำงานของโหนดบางส่วน
- **Dense Layer**: เลเยอร์สุดท้ายที่มีฟังก์ชันกระตุ้นแบบ Softmax เพื่อทำนายและให้ผลลัพธ์ออกมาเป็นความน่าจะเป็นของแต่ละคำในพจนานุกรม

จากนั้นทำการฝึกสอนโมเดล (Train) เป็นจำนวน 15 Epochs

### 4. การบันทึกโมเดลและการสร้างข้อความ (Save & Text Generation)
- เมื่อโมเดลเรียนรู้เสร็จสิ้น จะทำการบันทึกโมเดลเป็นโฟลเดอร์ `9arm_model_v1` และบันทึก Tokenizer เป็นไฟล์ `tokenizer.pickle` สำหรับนำไปใช้งานต่อ
- ฟังก์ชัน `generate_9arm_text` จะรับคำตั้งต้น (Seed text) และใช้โมเดลทำนายคำถัดไปมาต่อท้ายประโยคเรื่อยๆ ตามจำนวนคำที่ต้องการ จนได้ข้อความใหม่ที่มีความคล้ายคลึงกับสไตล์การพูดดั้งเดิมของข้อมูล
